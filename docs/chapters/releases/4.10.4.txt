Version 4.10.4
==============

Released: February 27, 2026

Summary
-------

Version 4.10.4 is a maintenance release focused on stability and operational polish. It backports a set of improvements from the 4.11 line, improves performance when hashing large files, adds administrative tooling for managing frontend dependencies, smooths out navigation in the Settings UI, and refreshes the project's container images and Python dependencies.


Highlights
----------

- Faster, more predictable hashing for large files via a shared chunked hashing utility with a tuned default block size.
- New management command for removing JavaScript and Google Font dependencies.
- Quality-of-life improvements across Settings and list pagination.
- Lower log verbosity to reduce noise in production.
- Updated Docker base images and dependency versions.


Changes
-------


Administration and developer tooling
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

- Added the `dependencies_uninstall` management command to remove installed JavaScript and Google Font dependencies.
- Renamed the make target `requirements-generate` to `python-requirements-generate` to better reflect its scope.
- Included backported makefile and GitLab CI improvements to streamline builds and pipelines.
- Renamed internal `DOCKER_ELASTIC_` references to `DOCKER_ELASTICSEARCH_` for consistency and clearer configuration.


Core
^^^^

- Added a shared chunked hashing utility to process large files efficiently without loading them fully into memory.
- Set the default hashing block size to 65536 bytes to reduce overhead and better match typical I/O and CPU behavior.
- Moved literals to `mayan/literals.py` to avoid circular import issues.
- Reduced default logging verbosity to improve operational signal-to-noise.
- Added `libfuse2` as a production dependency to improve compatibility in environments requiring FUSE libraries.


User interface
^^^^^^^^^^^^^^

- Added thousand separators to pager counters to make large result counts easier to read.
- Improved Settings return navigation and fixed a duplicated return link when editing settings.
- Added or improve the help text of several settings:

  - `DOCUMENTS_DISPLAY_HEIGHT`
  - `DOCUMENTS_DISPLAY_WIDTH`
  - `DOCUMENTS_PREVIEW_HEIGHT`
  - `DOCUMENTS_PREVIEW_WIDTH`
  - `DOCUMENTS_PRINT_HEIGHT`
  - `DOCUMENTS_PRINT_WIDTH`
  - `SIGNATURES_BACKEND_ARGUMENTS`
  - `STORAGE_DOWNLOAD_FILE_STORAGE_ARGUMENTS`

Documents
^^^^^^^^^

- Prevented duplicate favorite entries when a user favorites the same document multiple times.


Indexing
^^^^^^^^

- Strengthened cleanup of empty index instance nodes to keep index structures compact and consistent.


Templating
^^^^^^^^^^

- Corrected the template tag name from `math_substract` to `math_subtract` while keeping a temporary alias for backwards compatibility.


MIME type detection
^^^^^^^^^^^^^^^^^^^

- Switched MIME type backends from sh to Python subprocess to reduce external shell dependency and improve portability.


Search settings
^^^^^^^^^^^^^^^

- Updated `SEARCH_STORE_RESULTS_DEFAULT_VALUE` to use explicit choices for clearer configuration.
- Changed `SEARCH_MATCH_ALL_DEFAULT_VALUE` to a boolean and also exposed it as a choice field for consistent Settings UI handling.


Docker
^^^^^^

- Updated Docker image tags for Debian, Elasticsearch, PostgreSQL, RabbitMQ, Redis, and Traefik to pick up upstream fixes and security updates.
- Update Docker image tags:

  - Debian from debian:12.12-slim to debian:12.13-slim
