Version 4.11
============

Released: February 25, 2026


Summary
-------

Version 4.11 introduces major changes to the underlying systems that power Mayan EDMS.

This release moves the codebase from Django 4.2 to Django 5.2. The base image is also changed from Debian 12 to Debian 13. These two changes alone ensure Mayan EDMS 4.11 and later utilize core products that are fully supported for years to come.

This release also adds ARM64 Docker image support, allowing Mayan EDMS to run once again on single board computers as it did for a large portion of its early history.

The settings system has been completely rewritten to be more streamlined, support data types, improve serialization, and support more key value stores including vendor and cloud platform ones.

The CI, Docker build processes, error logging, and configuration handling also saw significant changes and improvements.

Dependencies were streamlined too, with many dependencies being updated. Deprecated support for some older technologies was removed.

Many internal subsystems have been reorganized for improved determinism, performance, scalability, and maintainability.

This release contains several changes that may require administrator attention. Review the sections below before upgrading.

The release of version 4.11 begins the sunsetting of version 4.6, which is now approaching end-of-life. Likewise, online translation files for version 4.5 have been deleted. Only offline translations are possible for version prior to 4.6.

Changes
-------


Backend
^^^^^^^

- Made the Celery application class configurable via the `MAYAN_CELERY_CLASS` setting.
- Disabled caching of `get_config_file_content` since it is only used during startup.
- Moved several modules to reduce circular dependencies.
- Replaced the `sh` library with Python `subprocess` in MIME type detection backends.
- Deprecated Oracle database support has been removed.
- Added a reusable block hashing function optimized for large files.


Dependencies
^^^^^^^^^^^^

- Updated to Django 5.2 LTS.
- Updated numerous Python libraries including Sphinx, Elasticsearch client, celery, gunicorn, redis, jsonschema, and wheel (addressing CVE-2026-24049).
- Updated Fancybox to 3.3.5.
- Added the management command `dependencies_uninstall` to remove JavaScript and Google Font dependencies.


Docker and Deployment
^^^^^^^^^^^^^^^^^^^^^

This release introduces extensive improvements to the build system and Docker-based deployments:

- Updated the base Docker image to Debian 13.3-slim.
- Added support for building ARM64 Docker images.
- Added support for building multiarch Docker images.
- Significant refactor of makefiles and CI configuration, including:
  - Splitting Docker makefiles, removing redundancy, reorganizing targets.
  - Adding AMD64 and ARM64 Docker image build jobs.
  - Improved caching for APK, PIP, and Docker BuildKit.
  - Normalization of environment variable usage in Docker scripts.
  - Addition of new variables such as `DOCKER_USER_USERNAME` (default `mayan`), used for generating containers and entrypoints.
- Updated Docker images for Elasticsearch, PostgreSQL, Python, RabbitMQ, Redis, Traefik, and Docker DinD.
- Improved Docker testing targets and PostgreSQL test container health checks.
- Entry point logic has been updated, including moving user switching to `entrypoint.sh` and adding `--chdir` to Gunicorn to avoid permission issues.
- Docker Compose was updated to align with the new entrypoint behavior.


Settings
^^^^^^^^

This version includes a major refactor of the settings subsystem:

- Settings now include first-class data type support and operate through a domain-based architecture that allows flexible backends depending on deployment needs.
- Settings are now immutable and are pre-cached on system startup to ensure deterministic access.
- Serialization responsibilities have moved from clusters to individual domains.
- Cluster-level cache invalidation now cascades through namespaces and settings.
- Added support for removing namespaces from clusters, and domain entries are now cleaned up when settings are removed.
- Setting pending value detection and revert are now fully reentrant, multi-user safe, and thread-safe.
- Backups of configuration files are no longer created during tests.
- Settings help text has been either added or expanded.
- Numerous settings that were previously string-based are now validated as integers. This affects pagination, document dimensions, caching, preview sizes, zoom levels, search limits, and more.
- Several search UI settings have been updated, including converting `SEARCH_STORE_RESULTS_DEFAULT_VALUE` and `SEARCH_MATCH_ALL_DEFAULT_VALUE` to choice fields and updating the default of the latter to a boolean.
- Added normalization helpers for environment variable construction, including the proto setting `MAYAN_ENVIRONMENT_VARIABLE_PREFIX`.


Templating
^^^^^^^^^^

- Renamed the incorrect template tag `math_substract` to `math_subtract`. The old name remains as a temporary alias but will be removed in a future minor version.
- Improved value handling, added decimal support, expanded help texts.
- Added new filters and tags including `math_ceil`, `math_floor`, `math_integer`, `math_maximum`, `math_minimum`, `math_percent`, and `math_round`.


User Interface
^^^^^^^^^^^^^^

- Improved navigation in settings views.
- Prevented creation of duplicate favorite entries.
- Reinforced pruning of empty index instance nodes.
- Added subtitles to object error log views showing the number of retained entries.
- Added thousand separators to pager values for improved readability.


Error Logging
^^^^^^^^^^^^^

- Error log messages are now more descriptive and translatable.
- To preserve debug information, error logs are no longer cleared automatically when operations succeed. Existing logs must be cleared manually.
- Increased the default retention per object from 3 to 15 entries.


Miscellaneous
^^^^^^^^^^^^^

- Added `__pycache__` to the git ignore list.
- The smart settings app views module was split and reorganized.
- The makefile target `requirements-generate` was renamed to `python-requirements-generate`.
- Logging output was reduced.
- `DOCKER_ELASTIC_` internal references were renamed to `DOCKER_ELASTICSEARCH_`.


Upgrade Notes
-------------

- Download the latest Docker Compose file or update your existing one to support the new service argument layout.
- Several settings now enforce integer values. Existing configurations using strings must be updated.
- Template authors using the `math_substract` tag should switch to `math_subtract` before the alias is removed in a future release.
- Custom deployments depending on Oracle must migrate to a supported database backend.
- Review CI customizations for compatibility with the new makefile layout, variables, and entrypoint changes.


Deprecations
------------

- `math_substract` tag added as a backward compatible alias will be removed in a future minor version.
